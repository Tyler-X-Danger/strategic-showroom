The Flow
How a query becomes an answer
No dashboards, no tickets, no waiting. The agent handles everything between the question and the insight.
1
Marketer asks a question in Slack
@mention the bot with any plain-English question about buyer segments, property types, market trends, or specific zip codes. No SQL, no syntax, no training required.
"@agent — where are First-Time Buyers growing fastest right now?"
2
The agent figures out what data to pull
GPT-4o reads the intent and selects the right combination of tools — automatically. A single question may trigger multiple data calls in parallel. The user never sees this complexity.
Runs: top markets → market context for each top zip → chart generation
3
Data is fetched from the intelligence layer
Tool calls route to purpose-built microservices — a data server and a visualization server — running as subprocesses via the Model Context Protocol. Today backed by DuckDB; tomorrow, Snowflake.
5 tools across 2 services: growth, trends, property types, market context, charts
4
A formatted answer — and a chart if needed — lands in the channel
GPT-4o synthesizes all the data into a scannable Slack message: top zip codes, growth rates, average sale prices, days-on-market, and a key insight. Trend charts are generated and uploaded automatically.
Text summary + PNG chart uploaded directly to the Slack channel

Out-of-Scope Handling
When the agent can't answer — it files a ticket
Questions that fall outside the agent's current toolset shouldn't disappear into a Slack thread. The agent can be wired to JIRA to route them automatically.
⚡ Today — agent responds with
·An honest "I can't answer that yet" message with context on what data or capability is missing.
·A suggestion for how to rephrase or what the data team would need to build to support it.
🔮 With JIRA integration — agent would
·Auto-create a JIRA ticket in the correct project with the original question as the description.
·Assign it to the right data team member based on request type (new dataset, new metric, new segment).
·Reply in Slack with the ticket link so the requester has a paper trail — no follow-up needed.
What's Next
Where this goes with the right setup
These capabilities are all architecture-compatible with what's already built — each one is an additive layer, not a rewrite.
❄️
NextLive Snowflake Connection
Swap the DuckDB mock for a real Snowflake connection — no code changes to the agent layer. The data server is already designed for this. Every query the prototype runs today will run against production data immediately.
🗂
NextJIRA Auto-Ticketing
When a question falls outside current capabilities, the agent creates a structured JIRA ticket in the data team's backlog — with the question, requester, channel, and priority — and replies with the ticket link in Slack.
📬
NextData Team Fielding
The agent becomes the first point of contact for data requests. It classifies intent, routes to the right team or tool, and only escalates what actually needs a human. Reduces ad-hoc Slack pings to the data team.
📅
FutureScheduled Market Briefings
A weekly summary — top growing markets, notable shifts, any hotness score changes — pushed automatically to a designated Slack channel every Monday morning. Zero manual work.
🚨
FutureMarket Alert Subscriptions
Marketers subscribe to triggers: "alert me when Charlotte First-Time Buyer growth crosses 50%" or "notify me if Luxury Estate DOM drops below 30 days." Alerts fire in Slack the moment the threshold is crossed.
📋
FutureCampaign Brief Generation
Turn a market insight query into a formatted campaign brief — target segment, top zip codes, recommended property types, pricing context, and suggested creative angle — ready to hand off to the campaign team or export as a doc.
🔗
FutureCRM & Audience List Push
After identifying the top markets for a segment, the agent pushes those zip codes directly into a Salesforce or HubSpot audience segment — no copy/paste, no CSV exports, no manual list builds.
💬
FutureCross-Channel Distribution
One query in one channel, results posted to multiple stakeholder channels automatically — e.g., an Investor markets update goes to the acquisitions channel, the marketing channel, and the exec digest channel simultaneously.
